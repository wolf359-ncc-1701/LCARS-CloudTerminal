# V0.75 Screenshot Regression Analysis

This document records the visual defects identified from the user's V0.7 screenshots. V0.75 is a corrective polish pass, not a new feature pass.

## Scope

V0.75 must fix specific visible regressions before V0.7 can be considered style-stable:

- Inconsistent typography in log and module surfaces.
- Command/Actions naming residue and title truncation.
- Energy module's unclear core diagram.
- Poor contrast on cyan/orange/gray panels.
- Missing/cut top-left LCARS elbow corner.
- Right rail menu items changing size and alignment across modes.

Do not add new application features. Do not push to GitHub until the user explicitly asks.

## Screenshot Findings

### 1. Log Typography And Wrapping

Affected screenshots:

- Log text appears as a different, softer font from the rest of the LCARS UI.
- Narrow panels wrap each log sentence into many short vertical lines.
- Wider panels preserve the sentence but still feel typographically inconsistent.

Likely surfaces:

- `MemoryView`
- event log blocks inside `CommandView` / Actions telemetry log
- shared `.event-log` CSS
- `.type-body-log`

Required direction:

- Use a consistent LCARS typography role for short log entries.
- For dense prose, use a readable fallback only when it is a true paragraph.
- Avoid 3-column event rows inside narrow nested panels.
- Add compact log variants for side panels.
- Prevent single sentence logs from turning into vertical word stacks.

Acceptance:

- Log lines in side panels remain readable.
- Log text no longer looks like a different app.
- Narrow panels use one-line or two-line records, not 5-8 line word stacks.

### 2. Energy Core Is Visually Meaningless

Affected screenshot:

- The Power/Energy module shows a large orange panel with a black rounded cavity and a glow.
- It reads as an abstract hole rather than an energy system.
- Labels are scattered and low contrast.

Likely surface:

- `EnergyView`
- `.warp-core`
- energy board CSS

Required direction:

- Replace the unclear "warp core" blob with an original LCARS Cloud energy-bus diagram.
- Use a meaningful structure such as:
  - central power bus,
  - vertical capacitor stack,
  - distributed room/device load rails,
  - breaker grid,
  - glowing flow path,
  - numeric load gates.
- Keep it original and not a Star Trek ship/warp-drive copy.
- Use clear labels and contrast.

Acceptance:

- The Energy view looks like a power distribution / smart-home energy bus.
- Labels are readable.
- The large orange block no longer dominates without function.

### 3. Color Contrast Is Too Weak

Affected screenshots:

- Cyan panels use gray/cyan text with poor contrast.
- Orange panels use pale text that disappears.
- Gray panels use low-contrast labels.
- Some meters and values are clipped or hidden against same-value backgrounds.

Likely surfaces:

- `HomeView`
- `EnergyView`
- `LcarsActionButton`
- `LcarsParameterRow`
- `.room-card-content`
- `.energy-board`
- `.lcars-element` color contrast rules

Required direction:

- Enforce text-on-color tokens:
  - cyan panels use `--text-on-cyan`.
  - orange panels use `--text-on-orange`.
  - dark panels use `--gray-white` or `--cyan-bright`.
- Avoid gray text on cyan/orange unless decorative and nonessential.
- Meter values must remain visible on all panel colors.

Acceptance:

- LIVING DECK / WORK BAY / SLEEP QUARTERS / ENTRY LOCK cards are readable at a glance.
- Energy load labels and numbers are readable.
- Orange warning panels still look LCARS but do not wash out text.

### 4. Top-Left LCARS Elbow Missing/Cut Corner

Affected screenshot:

- The upper-left corner rail appears to have a missing or cut block.
- The curved elbow does not look like a continuous LCARS frame.

Likely surfaces:

- `LcarsElbow`
- left rail header in `App.tsx`
- `.left-rail`
- `.lcars-app` padding / overflow interaction

Required direction:

- Fix the top-left elbow as a continuous shape.
- Avoid clipping the curved segment at the app boundary.
- Prefer pure CSS/SVG geometry that preserves the inside black void and outside rounded rail.
- The elbow should read like one connected LCARS element, not separated rectangles.

Acceptance:

- At 1440px and 1920px, the top-left rail has no missing block.
- At 390px and 768px, it either scales, hides cleanly, or remains continuous.

### 5. Top-Right Title Still Ellipsizes

Affected screenshots:

- `LCARS CLOUD T...`
- `COMMAND MOD...`
- Title area still truncates despite the prior fix.

Likely surfaces:

- `App.tsx`
- `.top-rail`
- `.safe-title-zone`
- `.right-rail`

Required direction:

- The title must not use ellipsis in desktop layouts.
- Rename mode title display to shorter safe labels:
  - Bridge: `LCARS CLOUD`
  - Home: `HOME`
  - Energy: `ENERGY`
  - Memory: `MEMORY`
  - Actions: `ACTIONS`
- Put secondary text (`SYSTEM INDEX / AL-52169`) into a separate right-side label, not in the same title collision zone.
- On narrow desktop, reduce font size before ellipsis.
- On mobile/tablet, allow two-line title if needed.

Acceptance:

- No desktop screenshot shows title ellipsis.
- Title and system index do not collide.
- User-facing label says `ACTIONS`, not `COMMAND MODULE`.

### 6. Right Rail Menu Size Changes Across Modes

Affected screenshots:

- Right-side menu entries appear with different heights or active block sizes across modes.
- The active item sometimes creates awkward bottom radius or huge orange block.
- Quick Actions area can overflow or clip labels, e.g. `RESUME NORMAL`.

Likely surfaces:

- `.right-rail`
- `.right-stack`
- right rail `LcarsElement` styles in `App.tsx`
- `.command-hints`

Required direction:

- Right menu items must have fixed height and stable layout across all modes.
- Active state changes color only, not size.
- Text should be right-aligned or consistently centered by design, not accidentally shifted.
- Quick action buttons need enough width, no clipping.
- Avoid inline `borderRadius: 0` for all menu items if it fights LCARS shape grammar.

Acceptance:

- Right rail has stable item dimensions when switching modes.
- `BRIDGE`, `HABITAT`, `POWER`, `MEMORY`, `ACTIONS` all align consistently.
- `RESUME NORMAL` and `CINEMA MODE` fit without clipping.

## V0.75 Design Rule

V0.75 should be less ambitious than V0.7 but more exacting:

- Fix visible defects.
- Reduce accidental complexity.
- Do not add a new mode.
- Do not introduce more colors.
- Do not redesign all pages from scratch.
- Keep the style between Titan DS density and the user's PPT-derived roundness.

## Must Not Do

- Do not push to GitHub.
- Do not merge into `main`.
- Do not add external assets.
- Do not reintroduce command text input.
- Do not remove audio support.
- Do not make the UI a direct clone of the user's PPT.
