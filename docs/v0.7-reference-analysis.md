# V0.7 Reference Analysis

This document summarizes the V0.7 design inputs from user feedback, current V0.6 screenshots, the user's PPT reference, exported PPT animation videos, and user-provided response audio.

User-provided local references:

- `C:\Users\user\Desktop\LCARS\LCARS.pptx`
- `C:\Users\user\Desktop\LCARS\LCARS.mp4`
- `C:\Users\user\Desktop\LCARS\LCARS1.mp4`
- Current V0.6 screenshots supplied in chat.
- User-provided audio files copied into `src/assets/audio/`.

PPT structure:

- 8 slides.
- Includes LCARS-like media frame pages, memory/credit matrix pages, sequence start/end pages, large black negative space, rounded rails, capsule rows, numeric clusters, and camera/archive metadata.
- Extracted text includes patterns such as `Initializing`, `Accessing local database`, `Sequence Ready`, `MEDIA 310`, `MEMORY`, `VSL MATRIX`, `OPS 10A`, `INDEX`, `PARAMETERS`, and dense numeric sequences.

Video metadata from Windows Shell:

- `LCARS.mp4`: 3840x2160, 62.5 fps, approximately 28 seconds.
- `LCARS1.mp4`: 3840x2160, 62.5 fps, approximately 42 seconds.

Interpretation:

- The animation target is high-refresh and crisp.
- Motion should be segmented, mechanical, and precise rather than slow web-app easing.
- Small elements should activate in phases: rail reveal, text decode, matrix fill, status blink, then stable idle motion.

## Current V0.6 Problems To Fix

User-observed issues:

- Some text still uses fallback typography and breaks the LCARS illusion.
- Tabs are too rectangular and generic.
- Some modules are misaligned or visually collide.
- Some color combinations are too close in value and become hard to read.
- The top-right title overflows and overlaps tabs/module areas.
- The command input mode feels unnecessary; direct action buttons are enough.

Code-level implication:

- Font application must be tokenized and broad, but not blindly forced onto body prose.
- The shell needs a safe title zone with max width, wrapping/scale rules, and no overlap with tabs.
- Tabs need LCARS-shaped caps, not generic rectangular buttons.
- Layout must be governed by CSS classes, not inline grid rules.
- The current Command view should become an Action Matrix view.

## PPT-Derived Design Direction

The user's PPT reference has a different strength than Titan DS:

- It uses large black negative space confidently.
- It prefers fewer, larger LCARS forms rather than dense dashboard clutter everywhere.
- It uses strong rounded elbows and capsule ends as the main visual identity.
- It places labels at the edge of shape systems, not always inside centered cards.
- It often pairs one large media/module frame with a side stack of parameter pills.
- It uses numeric and code text as atmosphere, not as explanatory copy.

V0.7 should combine:

- Titan DS density and operational feel.
- User PPT's cleaner black-space rhythm and rounder LCARS forms.
- The current app's cloud/home control concept.

Do not remake V0.6 exactly like the PPT. Use the PPT as a shape and motion reference, not as a template clone.

## Shape Grammar

V0.7 should move from square cards toward these primitives:

- **Elbow rail**: thick L-shaped or C-shaped rail with one large rounded end.
- **Capsule strip**: row item with a fully rounded left or right terminal.
- **Inset window**: black content cavity inside a colored LCARS frame.
- **Segment stack**: vertical rectangles separated by black gutters, often attached to a rounded top segment.
- **Pill parameter row**: label capsule on the left, numeric cluster on the right.
- **Long rail plus terminal block**: extended horizontal rail ending with a square or capsule block.
- **Data matrix**: low-contrast grid of small rounded rectangles, changing opacity or fill.
- **Decode text row**: short fake numeric strings that appear/disappear or slide in.

Avoid:

- Generic equal-radius cards.
- Plain square tabs.
- Centered marketing-style headings.
- Large text blocks explaining features.

## Typography Rules

Primary display font:

- `"LCARS Microgramma"`

Use it for:

- Mode labels.
- Rail numbers.
- Module names.
- Button/action labels.
- Numeric readouts.
- Short system status phrases.

Avoid it for:

- Long paragraph text.
- Dense event messages where readability matters.

Every major UI label should have a deliberate text role:

- `display-title`: large system title, uppercase, constrained width.
- `module-label`: short tab/action label.
- `numeric-readout`: large number or code.
- `micro-code`: decorative numbers and fake paths.
- `body-log`: readable event text, allowed to use a narrower fallback.

The V0.7 implementation should add CSS utility classes for these roles instead of one huge selector list.

## Color And Contrast Rules

The current V0.6 sometimes places gray text on mid gray or cyan text on cyan, which weakens readability.

Rules:

- Text on cyan panels should usually be near black or deep navy.
- Text on orange panels should be near black or very dark red.
- Gray panels need light gray text only when the gray is dark enough.
- Big active tabs can use cyan/orange fill, but inactive tabs should stay dark and readable.
- Top-right title must not share the same z-space as tabs.
- Avoid large same-value color blocks next to each other.

Introduce semantic tokens:

- `--text-on-cyan`
- `--text-on-orange`
- `--text-muted`
- `--panel-border`
- `--safe-title-max`

## Navigation And Actions

V0.7 should remove the typed command interface as a primary mode.

Replace user-facing `COMMAND` with:

- `ACTIONS`
- `SCENES`
- or `EXECUTE`

Recommended: `ACTIONS`.

Behavior:

- Users click action buttons directly.
- No text input needed for V0.7.
- Action buttons should trigger scenes, alerts, normal mode, audio toggle, display modes, and module shortcuts.
- The old command parser may remain internally if useful, but it should not be the primary UI.

## Animation Direction

From the PPT videos:

- Use fast segmented reveals.
- Prefer 120ms-360ms micro-steps.
- Avoid slow full-panel fades as the main effect.
- Animate rails, text, and matrices separately.
- Idle motion should be subtle: dots, matrix flicker, scanner ticks, tiny bar pulses.

Needed primitives:

- `rail-snap-in`
- `capsule-slide-in`
- `matrix-decode`
- `text-decode`
- `status-dot-wave`
- `media-frame-open`
- `parameter-row-lock`
- `action-confirm-flash`

All motion must respect `prefers-reduced-motion`.

## V0.7 Acceptance Focus

V0.7 is not about adding features. It is about defining the visual language.

Pass criteria:

- The interface no longer feels like square dashboard cards.
- The top-right title cannot overlap tabs.
- Every main mode uses consistent Microgramma labels.
- Action mode uses direct buttons, not typed commands.
- Audio is mapped and usable through the app audio setting.
- Screens at 390px, 768px, 1440px, and 1920px avoid text collisions.
- The visual style sits between Titan DS density and the user's PPT roundness/negative-space rhythm.
