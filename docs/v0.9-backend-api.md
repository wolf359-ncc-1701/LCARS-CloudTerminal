# V0.9 Read-Only Archive API

V0.9 adds a local-only backend service for the Memory module. It is intentionally read-only: it can expose project metadata, text files, search results, and TNG manual markdown chapters, but it cannot write, delete, move, or rename files.

## Run

```powershell
npm run api
```

The service listens on:

```text
http://127.0.0.1:8787
```

Vite proxies `/api` to this service during `npm run dev`.

## Endpoints

- `GET /api/health`
  - Returns service status and workspace root.
- `GET /api/files/tree?path=.`
  - Returns a recursive read-only project tree.
  - Excludes `.git`, `node_modules`, `dist`, `external`, `tmp`, and `output`.
- `GET /api/files/meta?path=src/app/App.tsx`
  - Returns file metadata.
- `GET /api/files/read?path=src/app/App.tsx`
  - Returns UTF-8 text content for readable project files up to 1 MB.
  - Binary or oversized files return metadata with `content: null`.
- `GET /api/search?q=Memory`
  - Searches filenames and readable text snippets inside the workspace.
- `GET /api/archive/manuals`
  - Lists local archive collections.
- `GET /api/archive/manuals/tng-technical-manual-cn/outline`
  - Returns the TNG manual manifest and outline.
- `GET /api/archive/manuals/tng-technical-manual-cn/chapter?file=computer-systems.md`
  - Returns one TNG manual markdown chapter and a heading outline.

## Security Model

- All workspace file paths are resolved through a root guard.
- Absolute paths and `..` escape attempts are rejected.
- Only `GET` and `OPTIONS` are supported.
- No write endpoints exist in V0.9.
- Manual chapter access is restricted to `external/stcn-docs/docs/docs/tng-technical-manual/*.md`.

## Frontend Contract

The Memory UI should treat TNG manual chapters and project files as two source groups within one archive manager, not as unrelated tabs.

Do not embed the generated PDF in the UI. The project reader should render markdown and text through LCARS-native panels.
